使命

社交连接属于街道，而非信息流

大型社交网络有一个根本性问题：它们为注意力而优化，而非为连接。它们的商业模式是广告，工具是算法，结果是躲在发光屏幕后面的孤立。

MapRaiders走的是相反的路。社交网络就是地图本身。你看到的不是精心策划的内容——它是真实的。谁在你的城市活跃，谁占领领土，谁留下回声，谁创建任务：这一切都发生在外面，在世界上，对同样在外面的每个人都可见。

没有任何资料会因为算法想要最大化互动而被推荐给你。你与其他玩家相遇，是因为你们字面意义上在同一个地方。这种连接无法伪造——这正是重点所在。

创立故事

2025年创立——从一个简单的问题开始

如果地图就是社交网络呢？不是一张添加了社交功能的地图，而是一个以地理位置为每次互动基础的平台。

MapRaiders于2025年在Scafa Investments LLC（美国佛罗里达州卡特勒贝）旗下创立。契机：Ingress或Pokémon GO等现有位置游戏表明，人们愿意为了数字内容在现实世界中行走。但没有任何一款游戏迈出下一步——将地图作为主要社交媒介来对待。

团队从一个核心理念出发：领土作为个人声明。当你在自己城市占据一块区域时，那是一个声明。它是可见的。它需要真实的存在。正是在那里，有着任何算法都无法制造的社交信号。

技术

为真实GPS性能而构建

移动端 — React Native / Expo

MapRaiders应用使用Expo SDK上的React Native开发。全面采用TypeScript，使用Zustand进行状态管理，采用隐私优先设计的设备端GPS追踪。没有用户明确操作，位置数据不会离开设备。

服务器 — Node.js + PostgreSQL + PostGIS

后端栈在Node.js、Express和TypeScript上运行。带PostGIS扩展的PostgreSQL 16实现高性能空间查询——多边形重叠、邻近检查、实时领土合并操作。

实时 — WebSocket + Redis

所有实时事件（领土攻击、回声检测、事件更新）通过WebSocket连接流动。Redis处理会话管理、速率限制以及服务器实例之间的pub/sub。

隐私优先——无追踪商业模式

MapRaiders不为广告网络收集数据。GPS数据仅用于游戏机制。无第三方追踪，无数据转售，应用中无广告SDK。

我们的价值观

MapRaiders永远不会成为的东西

没有算法

你在地图上看到的是未经过滤的现实。没有推荐引擎决定你看哪些内容——只有地理和真实人的活动。

永不投放广告

MapRaiders永远不会引入广告支持的模式。未来收入仅来自创作者的可选打赏和提升游戏体验的高级功能——而非操纵用户。

内容所有权归用户

任务、回声、挑战和领土属于创建它们的玩家。MapRaiders是平台，不是内容所有者。你的创作是你的。
