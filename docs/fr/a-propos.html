Mission

La connexion sociale appartient à la rue, pas au fil d'actualité

Les grands réseaux sociaux ont un problème fondamental : ils optimisent pour l'attention, pas pour la connexion. Leur modèle économique est la publicité, leur outil est l'algorithme, et leur résultat est l'isolement derrière des écrans lumineux.

MapRaiders adopte l'approche inverse. Le réseau social, c'est la carte elle-même. Ce que vous voyez n'est pas curé — c'est réel. Qui est actif dans votre ville, qui tient des territoires, qui laisse des échos, qui crée des quêtes : tout cela se passe dehors, dans le monde, visible pour tous ceux qui sont également dehors.

Aucun profil ne vous est recommandé parce qu'un algorithme veut maximiser l'engagement. Vous rencontrez d'autres joueurs parce que vous êtes littéralement au même endroit. Ce type de connexion ne peut pas être simulé — et c'est précisément le but.

Notre histoire

Fondé en 2025 — avec une simple question

Et si la carte était le réseau social ? Pas une carte avec des fonctionnalités sociales ajoutées, mais une plateforme où la position géographique est le fondement de chaque interaction ?

MapRaiders a été lancé en 2025 sous Scafa Investments LLC (Cutler Bay, Floride, États-Unis). L'impulsion : les jeux de localisation existants comme Ingress ou Pokémon GO montrent que les gens sont prêts à se déplacer dans le monde réel pour des contenus numériques. Mais aucun n'a franchi l'étape suivante — traiter la carte comme le principal medium social.

L'équipe a démarré avec une idée centrale : les territoires comme déclarations personnelles. Quand vous tenez une zone dans votre ville, c'est une prise de position. Elle est visible. Elle exige une présence réelle. Et c'est précisément là que réside le signal social qu'aucun algorithme ne peut fabriquer.

Technologie

Conçu pour de vraies performances GPS

Mobile — React Native / Expo

L'application MapRaiders est développée avec React Native sur Expo SDK. TypeScript intégral, Zustand pour la gestion d'état, suivi GPS sur l'appareil avec une conception respectueuse de la vie privée. Aucune donnée de localisation ne quitte l'appareil sans action explicite de l'utilisateur.

Serveur — Node.js + PostgreSQL + PostGIS

Le backend fonctionne sur Node.js avec Express et TypeScript. PostgreSQL 16 avec l'extension PostGIS permet des requêtes spatiales haute performance — chevauchements de polygones, vérifications de proximité, opérations de fusion de territoires en temps réel.

Temps réel — WebSocket + Redis

Tous les événements en direct (attaques de territoires, détections d'échos, mises à jour d'événements) circulent via des connexions WebSocket. Redis gère la gestion des sessions, la limitation de débit et le pub/sub entre les instances de serveur.

Vie privée d'abord — pas de modèle de suivi publicitaire

MapRaiders ne collecte pas de données pour les réseaux publicitaires. Les données GPS sont utilisées exclusivement pour les mécaniques de jeu. Pas de suivi tiers, pas de revente de données, pas de SDK publicitaire dans l'application.

Nos valeurs

Ce que MapRaiders ne sera jamais

Pas d'algorithme

Ce que vous voyez sur la carte est la réalité non filtrée. Aucun moteur de recommandation ne décide quel contenu vous voyez — seulement la géographie et l'activité de vraies personnes.

Jamais de publicité

MapRaiders n'introduira jamais un modèle financé par la publicité. Les revenus futurs proviennent exclusivement de pourboires optionnels pour les créateurs et de fonctionnalités premium qui améliorent le gameplay — sans manipulation.

Propriété du contenu par les utilisateurs

Les quêtes, échos, défis et territoires appartiennent aux joueurs qui les ont créés. MapRaiders est une plateforme, pas un propriétaire de contenu. Vos créations sont vos créations.
