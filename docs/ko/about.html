미션

소셜 연결은 거리에 속하는 것이지, 피드가 아닙니다

대형 소셜 네트워크에는 근본적인 문제가 있습니다. 연결이 아닌 주목을 위해 최적화되어 있습니다. 비즈니스 모델은 광고이고, 도구는 알고리즘이며, 결과는 빛나는 화면 뒤의 고립입니다.

MapRaiders는 반대 방향을 택합니다. 소셜 네트워크는 지도 그 자체입니다. 당신이 보는 것은 큐레이션된 것이 아닙니다 — 그것은 현실입니다. 당신의 도시에서 누가 활동하는지, 누가 영토를 보유하는지, 누가 에코를 남기는지, 누가 퀘스트를 만드는지: 이 모든 것이 밖에서, 세상에서 일어나며, 밖에 있는 모든 사람에게 보입니다.

알고리즘이 참여도를 극대화하고 싶어서 프로필을 추천하는 일은 없습니다. 다른 플레이어와 만나는 건 문자 그대로 같은 장소에 있기 때문입니다. 이런 방식의 연결은 조작할 수 없습니다 — 그것이 바로 핵심입니다.

창업 스토리

2025년 설립 — 하나의 단순한 질문으로부터

지도가 소셜 네트워크라면 어떨까요? 소셜 기능이 추가된 지도가 아니라, 지리적 위치가 모든 상호작용의 기반이 되는 플랫폼으로서의 지도.

MapRaiders는 2025년 Scafa Investments LLC(미국 플로리다주 커틀러 베이) 하에 설립되었습니다. 계기: Ingress나 Pokémon GO 같은 기존 위치 기반 게임은 사람들이 디지털 콘텐츠를 위해 현실 세계를 걸어 다닐 의향이 있음을 보여줍니다. 하지만 어떤 게임도 다음 단계를 밟지 않았습니다 — 지도를 주요 소셜 미디어로 다루는 것입니다.

팀은 하나의 핵심 아이디어로 시작했습니다: 개인적 선언으로서의 영토. 당신이 도시의 한 구역을 보유할 때, 그것은 선언입니다. 눈에 보입니다. 실제 존재를 요구합니다. 그리고 바로 거기에 어떤 알고리즘도 만들 수 없는 소셜 신호가 있습니다.

기술

진짜 GPS 성능을 위해 구축

모바일 — React Native / Expo

MapRaiders 앱은 Expo SDK 위에서 React Native로 개발됩니다. 전면 TypeScript, 상태 관리에 Zustand, 프라이버시 우선 설계의 온디바이스 GPS 트래킹. 사용자의 명시적인 작업 없이는 위치 데이터가 기기를 벗어나지 않습니다.

서버 — Node.js + PostgreSQL + PostGIS

백엔드 스택은 Node.js, Express, TypeScript로 실행됩니다. PostGIS 확장 기능이 있는 PostgreSQL 16은 고성능 공간 쿼리를 가능하게 합니다 — 폴리곤 겹침, 근접 확인, 실시간 영토 병합 작업.

실시간 — WebSocket + Redis

모든 라이브 이벤트(영토 공격, 에코 감지, 이벤트 업데이트)는 WebSocket 연결을 통해 흐릅니다. Redis는 세션 관리, 속도 제한, 서버 인스턴스 간 pub/sub을 처리합니다.

프라이버시 우선 — 추적 비즈니스 모델 없음

MapRaiders는 광고 네트워크를 위한 데이터를 수집하지 않습니다. GPS 데이터는 게임 메카닉에만 사용됩니다. 서드파티 트래킹 없음, 데이터 재판매 없음, 앱 내 광고 SDK 없음.

우리의 가치

MapRaiders가 절대 되지 않을 것들

알고리즘 없음

지도에서 보는 것은 필터링되지 않은 현실입니다. 어떤 추천 엔진도 어떤 콘텐츠를 볼지 결정하지 않습니다 — 오직 지리와 실제 사람들의 활동만 있습니다.

광고는 영원히 없음

MapRaiders는 광고 기반 모델을 도입하지 않습니다. 미래 수익은 선택적 크리에이터 팁과 게임플레이를 조작이 아닌 향상시키는 프리미엄 기능에서만 나옵니다.

콘텐츠 소유권은 사용자에게

퀘스트, 에코, 챌린지, 영토는 그것을 만든 플레이어의 것입니다. MapRaiders는 플랫폼이지 콘텐츠 소유자가 아닙니다. 당신의 창작물은 당신의 것입니다.
